## 一、赛题核心需求

**赛题**：A5-景区导览服务AI数字人（出题企业：锐捷网络苏州）

### 游客交互侧（3 项）
| # | 功能 | 说明 |
|---|---|---|
| 1 | **多模态交互** | 语音输入 + 文本输入，数字人以语音、表情、口型同步回答 |
| 2 | **智能问答与讲解** | 准确回答景区历史、文化、景点特色等问题 |
| 3 | **个性化推荐** | 根据游客兴趣（如"喜欢自然风光"）推荐游览路线和讲解重点 |

### 管理后台侧（4 项）
| # | 功能 | 说明 |
|---|---|---|
| 4 | **知识库管理** | 管理员可上传/更新景区讲解词、文史资料、FAQ |
| 5 | **数字人形象管理** | 配置数字人外观、服装、声音 |
| 6 | **游客感受度报告** | 关注点分析、情感趋势、服务建议 |
| 7 | **数据大屏** | 服务人次、热门问答、满意度趋势 |

### 硬性指标
| 指标 | 要求 |
|---|---|
| 模型 | 至少 1 个多模态大模型 |
| 知识库 | 本地景区知识库，事实性问答准确率 ≥ 90% |
| 延迟 | 语音问答延迟 < 5 秒 |
| 口型/表情 | 自然度由专家评估 |

### 交付物
1. 源码或可执行文件
2. 产品部署和使用手册
3. 产品总体设计文档
4. 方案介绍 PPT
5. 演示视频（≤7 分钟）

### 评分权重
| 评分项 | 分值 |
|---|---|
| 功能完整度（多模态交互、智能问答、管理后台） | **40** |
| 技术与创新（数字人集成 15 + 大模型/知识库 15） | **30** |

## 二、技术选型

| 层级 | 技术 | 说明 |
|---|---|---|
| **渲染层** | LiveTalking + wav2lip | 2D 图片驱动口型同步，WebRTC 推流，RTX 4060 推理 |
| **大脑层** | Fay 框架 + Qwen Omni | 对话管理、RAG 知识库、弹幕/业务调度（目前仅用其 LLM 对接能力） |
| **ASR** | （待定） | 推荐 FunASR/Paraformer |
| **TTS** | edge-tts | 微软神经网络语音，免费在线 |
| **LLM** | Qwen Omni (通义千问全模态) | 多模态大模型（文本 + 图片），赛题硬性要求 ✅ |
| **RAG** | Chroma 向量库 + bge-m3 嵌入 | 景区知识库本地检索 |
| **管理后台** | FastAPI + Vue3 + ECharts | 知识库管理 / 形象管理 / 游客报告 / 数据大屏 |
| **存储** | SQLite + Chroma + 文件系统 | 轻量本地，零运维 |
| **部署** | Docker Compose | 一键编排 |
