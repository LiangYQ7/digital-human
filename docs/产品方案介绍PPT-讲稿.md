## 📄 第 5 页：核心技术 ① — RAG 知识检索

**标题**：三阶段 RAG 检索，确保回答准确率 ≥ 90%

**流程图**：

```
用户问题               阶段1              阶段2              阶段3
  │                    │                  │                  │

 有多高？"       向量检索              精选 3-4 条         结合上下文
                 top_k=12             最相关片段          生成回答
                   │                    │                  │
                   ▼                    ▼                  ▼
              ChromaDB              Qwen Omni          "灵山大佛通高
              142 个语义块           语义匹配            88 米，青铜铸造"
```

**数据支撑**：

- QA 测试集：51 题，覆盖 12 个维度

## 📄 第 6 页：核心技术 ② — 数字人渲染

**标题**：实时口型同步数字人

**技术栈**：

| 组件 | 技术 | 说明 |
|------|------|------|
| 口型推理 | wav2lip256 (GAN) | 根据音频特征生成口型画面 |
| 语音合成 | edge-tts | 微软神经网络 TTS，8 种音色 |
| 实时推流 | WebRTC | 浏览器原生支持，低延迟 |
| GPU 加速 | CUDA 12.8 | RTX 4060 Laptop，推理 60 FPS |

**性能数据**：
- 端到端延迟 < 3 秒（TTS 合成 + 口型推理 + WebRTC 传输）
- 支持多并发（5 路同时推流）
- 形象素材仅需 550 帧（~50MB），制作成本低

**配图**：数字人截图 + 帧率性能柱状图

## 📄 第 7 页：核心技术 ③ — 管理后台

**标题**：数据驱动的景区运营管理

**四大模块**：

| 模块 | 功能 | 技术 |
|------|------|------|
| 📊 数据大屏 | 服务人次、满意度趋势、热门问答 | ECharts 6 图表 |
| 📚 知识库 | 文档上传、自动分块、重建索引 | ChromaDB + jieba |
| 🎭 形象管理 | 音色切换、实时预览、文本驱动 | WebRTC + API |
| 📋 游客报告 | 情感分析、关键词、主题、时段、AI 总结 | jieba + LLM |

**分析能力**：
- jieba 分词 + 30+ 景区专有名词词典 → 有意义的词云
- 6 大主题分类（门票/交通/景点/餐饮/攻略/表演）
- 24 小时时段分布 → 运营排班参考
- AI 生成运营洞察建议
